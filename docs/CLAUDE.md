# Wolf Instruments - Claude Code Configuration

This file provides context for Claude Code when working in the Wolf Instruments project alongside Antigravity.

## Project Structure
- `docs/` - The Wolf Pacc Audio beat store website (Vanilla HTML/CSS/JS). Served via GitHub Pages.
- Root directory contains the Howling Wolves VST C++ source code and CMake build files.

## Workflow: Antigravity + Claude Code
- **Antigravity (Gemini)** is the primary architect. It handles planning, complex reasoning, generating `implementation_plan.md` files, and recording browser tests.
- **Claude Code** is the executor. It should follow the plans laid out in `implementation_plan.md` or `task.md` created by Antigravity, perform precise multi-file edits, and run terminal commands.
- When Antigravity requests Claude Code to perform an action, Claude Code should execute it aggressively, run necessary local tests, and report back by pushing to git or outputting to the terminal.

## Build Commands
- Website: No build step required (static HTML). To test locally, run `python3 -m http.server 8000` inside the `docs/` folder.
- VST Plugin: Built using CMake.
  ```bash
  mkdir -p build_v18
  cd build_v18
  cmake -G Xcode ..
  cmake --build . --config Release
  ```

## Development Guidelines
- **Website Development:** Maintain the dark premium aesthetic (deep blacks, cyan/teal accents). Do not use Tailwind; stick to the existing vanilla CSS in `docs/styles.css`. Keep URLs clean by placing `index.html` inside appropriately named folders (e.g., `docs/beats/index.html`).
- **VST Development:** Ensure compatibility with both Apple Silicon (M1/M2/M3) and Intel macOS, as well as Windows.
